# Getting Started with Jolli 

Welcome to Jolli! This guide will help you get up and running with Jolli, an AI-powered platform for creating, managing, and publishing documentation.

## What is Jolli?

Jolli is an intelligent documentation platform that helps teams:

- **Create AI-powered articles** - Generate and edit documentation with AI assistance
- **Build documentation sites** - Create and deploy Docusaurus or Nextra sites automatically
- **Integrate with GitHub** - Connect your repositories for seamless documentation workflows
- **Collaborate with AI** - Use the built-in chat to get help writing and improving content

## Prerequisites

Before you begin, make sure you have:

- A modern web browser (Chrome, Firefox, Safari, or Edge)
- A GitHub account (for repository integrations)
- Basic familiarity with Markdown (helpful but not required)

## Quick Start

### 1. Access Jolli

Navigate to your Jolli instance (e.g., `https://demo.jolli.ai` or your organization's URL).

### 2. Sign In

Click **Sign In** and authenticate with your GitHub account. Jolli uses GitHub OAuth for secure authentication.

### 3. Explore the Dashboard

After signing in, you'll see the main dashboard with several tabs:

- **Dashboard** - View recent jobs and activity
- **Articles** - Create and manage documentation articles
- **Doc Sites** - Create and deploy documentation websites
- **Integrations** - Connect GitHub repositories
- **Settings** - Customize your preferences

## Creating Your First Article

### Using the Articles Tab

1. **Navigate to Articles**
   - Click the **Articles** tab in the navigation bar

2. **Create a New Article**
   - Click the **New Article** button
   - Enter a title for your article
   - Choose a template (optional) or start from scratch

3. **Write Content**
   - Use the editor to write your content in Markdown
   - Format text using standard Markdown syntax
   - Preview your changes in real-time

4. **Get AI Assistance**
   - Click the **Chat** icon to open the AI assistant
   - Ask questions like:
     - "Help me write an introduction for this article"
     - "Improve the clarity of this section"
     - "Add code examples for this concept"

5. **Save Your Work**
   - Click **Save** to store your article
   - Your changes are automatically saved as drafts

## Creating a Documentation Site

### Setting Up Your First Site

1. **Navigate to Doc Sites**
   - Click the **Doc Sites** tab

2. **Create a New Site**
   - Click **Create New Site**
   - Fill in the site details:
     - **Site Name** - A unique name for your site
     - **Framework** - Choose Docusaurus or Nextra
     - **GitHub Organization** - Where the repository will be created

3. **Configure Deployment**
   - Jolli automatically creates a GitHub repository
   - The site is deployed to Vercel for hosting
   - You'll receive a live URL once deployment completes

4. **Add Content**
   - Add articles to your site
   - Organize content with folders and categories
   - Customize the site theme and navigation

## Connecting GitHub Repositories

### Setting Up Integrations

1. **Navigate to Integrations**
   - Click the **Integrations** tab

2. **Install GitHub App**
   - Click **Install GitHub App**
   - Select the repositories you want to connect
   - Authorize the Jolli app

3. **Use Repository Content**
   - Import code documentation from your repos
   - Generate API docs automatically
   - Keep documentation in sync with code changes

## Using the AI Chat Assistant

### Getting Help While You Work

The AI chat assistant is available throughout Jolli to help you:

1. **Open the Chat**
   - Click the **Chat** icon (usually in the bottom-right corner)

2. **Ask Questions**
   - Get writing suggestions
   - Request content improvements
   - Ask for technical explanations
   - Generate code examples

3. **Context-Aware Assistance**
   - The AI understands what you're working on
   - It can reference your current article or site
   - It provides relevant, specific suggestions

### Example Prompts

Try these prompts to get started:

- "Write an introduction explaining what [your product] does"
- "Create a step-by-step tutorial for [feature]"
- "Add troubleshooting tips for common errors"
- "Generate a code example showing how to [task]"
- "Improve the clarity of this paragraph"

## Understanding the Dashboard

### Monitoring Activity

The Dashboard tab shows:

- **Recent Jobs** - Background tasks like site deployments
- **Job Status** - See what's in progress, completed, or failed
- **Activity Feed** - Track changes across your projects

### Job Types

You'll see different types of jobs:

- **Site Creation** - Building new documentation sites
- **Content Generation** - AI-powered article creation
- **Repository Sync** - Updating content from GitHub
- **Deployment** - Publishing sites to hosting platforms

## Tips for Success

### Writing Great Documentation

1. **Start with an outline** - Plan your structure before writing
2. **Use clear headings** - Help readers navigate your content
3. **Include examples** - Show, don't just tell
4. **Keep it concise** - Shorter is often better
5. **Use the AI assistant** - Get suggestions and improvements

### Organizing Content

1. **Use consistent naming** - Make articles easy to find
2. **Create logical categories** - Group related content together
3. **Add metadata** - Use tags and descriptions
4. **Link between articles** - Help readers discover related content

### Collaborating with Your Team

1. **Share sites** - Give team members access to your documentation
2. **Use version control** - Track changes through GitHub integration
3. **Review together** - Use the chat to discuss improvements
4. **Establish conventions** - Agree on style and formatting standards

## Common Workflows

### Workflow 1: Creating Documentation from Code

1. Connect your GitHub repository in Integrations
2. Navigate to Articles
3. Use the AI to generate docs from your codebase
4. Review and refine the generated content
5. Publish to a Doc Site

### Workflow 2: Building a Knowledge Base

1. Create a new Doc Site
2. Plan your content structure
3. Create articles for each topic
4. Use the AI to expand and improve content
5. Deploy and share with your team

### Workflow 3: Maintaining Documentation

1. Monitor the Dashboard for updates
2. Review articles regularly
3. Use GitHub integration to stay in sync with code
4. Update and republish sites as needed

## Keyboard Shortcuts

Speed up your workflow with these shortcuts:

- `Ctrl/Cmd + S` - Save current article
- `Ctrl/Cmd + K` - Open AI chat
- `Ctrl/Cmd + P` - Preview article
- `Esc` - Close modals and dialogs

### Common Questions

**Q: How do I delete an article?**
A: Open the article, click the menu (three dots), and select Delete.

**Q: Can I export my content?**
A: Yes, articles are stored as Markdown and can be exported from GitHub.

**Q: How do I change my site's theme?**
A: Edit the site configuration in the Doc Sites tab.

**Q: Is my content automatically saved?**
A: Yes, Jolli auto-saves drafts as you work.
